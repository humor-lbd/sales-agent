# sales-agent 逐文件学习注释

这份文档是给“要学习并复现 `sales-agent`，但暂时不想一头扎进源码”的同学准备的。你可以把它当成一份按文件路径展开的讲解手册：先知道每个文件是做什么的，再决定哪些文件需要精读，哪些文件只要知道位置和作用就够了。

## 1. 先建立整体认识

`sales-agent` 是一个“销售数据分析 Agent”项目，整体可以拆成 4 层：

1. 后端接口层：用 FastAPI 对外提供登录、聊天、流式聊天、指标、测试工具接口。
2. Agent 编排层：用 LangGraph 把“读历史 -> Guardrails -> ReAct 决定是否调工具 -> 执行工具 -> 生成最终回答 -> 存记忆”串起来。
3. 数据与业务层：用 SQLAlchemy 访问 MySQL，再由 Service 和 Tool 层把数据库结果变成可回答用户的问题。
4. 前端展示层：用 Vue 3 + Element Plus + ECharts 做登录页、聊天页、图表渲染和指标面板。

如果你是第一次接触这类项目，建议阅读顺序是：

1. `README.md`
2. `app/main.py`
3. `app/api/routes.py`
4. `app/logic/agent.py`
5. `app/graph/` 整个目录
6. `app/logic/services.py` 和 `app/logic/tools.py`
7. `app/db/` 整个目录
8. `frontend/src/views/ChatView.vue`
9. `tests/` 目录

## 3. 根目录文件

### `.env`

作用：
项目本地运行时真正读取的环境变量配置文件。

关键点：
- 定义应用名、端口、JWT、MySQL、Redis、OpenAI、异常检测阈值等参数。
- 你现在的 Python 版数据库名也在这里配置。
- 这是“本地真实配置”，一般不要提交真实密钥到公共仓库。

学习建议：
复现项目时，先看这个文件，弄清楚项目到底连了哪些外部服务。

### `.env.example`

作用：
环境变量模板文件。

关键点：
- 给新同学或新环境提供一个可复制的配置骨架。
- 一般不放真实密码，只放字段名和示例值。

学习建议：
把它和 `.env` 对照着看，可以快速知道哪些配置是必须填的。

### `README.md`

作用：
Python 版项目的总说明文档。

关键点：
- 说明了项目用途、启动方式、数据库初始化方式和主要接口。
- 是你复现项目时第一份应该看的材料。

学习建议：
先看这里建立全局印象，再去看源码，会轻松很多。

### `requirements.txt`

作用：
后端 Python 依赖清单。

关键点：
- `fastapi`、`uvicorn`：Web 服务。
- `sqlalchemy`、`pymysql`：数据库访问。
- `redis`：缓存。
- `openai`、`langgraph`、`langchain-openai`：Agent 和模型调用。
- `pytest`：测试。

学习建议：
如果你不熟悉项目技术栈，这个文件能快速告诉你“这个项目用了什么框架”。

### `run_dev.ps1`

作用：
Windows 下的本地开发启动脚本。

关键点：
- 固定使用 `E:\develop\mini_anaconda\envs\dev\python.exe`。
- 用 `uvicorn` 的 `--reload` 模式启动后端，适合开发联调。

学习建议：
如果你在 Windows 上复现，这是最直接的启动入口。

## 4. 后端入口层

### `app/main.py`

作用：
后端服务真正的入口文件。

关键点：
- 创建 FastAPI 应用实例。
- 在生命周期启动时初始化数据库。
- 注册异常处理、请求日志和所有路由。

你读这个文件时要重点看：
- 应用是怎么创建的。
- 启动时做了哪些准备动作。
- 路由是从哪里挂载进来的。

学习建议：
把它理解成“总装配厂”，它自己业务不多，但决定项目怎么跑起来。

### `app/api/routes.py`

作用：
定义所有 HTTP 接口。

关键点：
- `/health`：健康检查。
- `/auth/login`、`/auth/logout`：登录与退出。
- `/agent/chat`：同步聊天。
- `/agent/chat/stream`：流式聊天。
- `/agent/session/{id}`：删除会话记忆。
- `/test/tool/*`：绕过 LLM，直接测工具能力。

你读这个文件时要重点看：
- 请求参数从哪里进来。
- `SalesAgent` 是怎么构造的。
- Tool 测试接口为什么能帮助调试。

学习建议：
如果你想先“跑通接口”，这里比 Graph 和 Tool 代码更适合作为入口。

## 5. `app/core`：基础设施层

### `app/core/config.py`

作用：
集中管理配置项。

关键点：
- 用 `pydantic-settings` 从环境变量读取配置。
- 提供数据库连接串等派生属性。

学习建议：
复现失败时，第一优先怀疑这里对应的环境变量有没有配对。

### `app/core/exceptions.py`

作用：
统一注册异常处理逻辑。

关键点：
- 把参数错误、OpenAI 错误、未知异常转成统一 HTTP 响应。
- 避免异常原样抛给前端。

学习建议：
这是“对外错误长什么样”的总入口。

### `app/core/logging.py`

作用：
配置日志并统计请求日志。

关键点：
- 初始化日志格式。
- 给 FastAPI 增加请求耗时记录。
- 把请求指标喂给 `metrics.py`。

学习建议：
调试接口慢、接口报错时，这个文件很重要。

### `app/core/metrics.py`

作用：
保存运行期指标快照。

关键点：
- 统计请求数量、数据库耗时、缓存命中、LLM 调用、工具调用。
- 支撑 `/ops/metrics` 接口。

学习建议：
如果你想知道“系统到底慢在哪”，这里是核心观测点。

### `app/core/redis_client.py`

作用：
集中创建 Redis 客户端。

关键点：
- 用缓存避免重复创建连接。
- Redis 没开时可以返回 `None`，让系统退化运行。

学习建议：
这是典型的“可选依赖”写法，适合学习项目如何优雅降级。

### `app/core/security.py`

作用：
处理登录鉴权和当前用户解析。

关键点：
- 生成和校验 JWT。
- 根据销售员信息构造当前用户对象。
- 提供路由依赖，控制哪些接口必须登录。

学习建议：
如果你不清楚“销售员为什么只能看自己的数据”，就来读这个文件和 `services.py`。

## 6. `app/db`：数据库层

### `app/db/bootstrap.py`

作用：
启动时初始化 Python 版自己的数据库。

关键点：
- 创建 Python 独立数据库。
- 执行 `schema.sql` 建表。
- 在业务表为空时执行 `data.sql` 导入演示数据。

学习建议：
你要复现数据库时，先看这里，再看 SQL 文件。

### `app/db/database.py`

作用：
创建 SQLAlchemy Engine、Session，并提供数据库依赖。

关键点：
- 全局维护引擎和会话工厂。
- 记录数据库查询耗时到指标模块。
- 让 FastAPI 路由通过依赖拿到 `db session`。

学习建议：
把它当成“数据库连接总闸门”。

### `app/db/models.py`

作用：
定义 ORM 模型。

关键点：
- `SalesRegion`：销售大区。
- `SalesRep`：销售员。
- `Product`：产品。
- `SalesOrder`：销售订单。
- `ChatMemory`：会话记忆。

学习建议：
数据库表结构和 Python 对象结构的映射关系在这里看最清楚。

### `app/db/repositories.py`

作用：
把常用数据库查询封装成 Repository。

关键点：
- 每个实体都有自己的查询方法。
- 订单仓库承载了排行、趋势、退款率等复杂聚合 SQL。
- 这是 Service 层真正依赖的“数据读取接口”。

学习建议：
如果你想理解“某个统计结果到底怎么查出来的”，这里最关键。

### `app/db/sql/schema.sql`

作用：
数据库建表脚本。

关键点：
- 定义五张核心表。
- 包含主键、唯一键和必要索引。

学习建议：
不懂 ORM 也没关系，先看 SQL 表结构，一样能理解业务模型。

### `app/db/sql/data.sql`

作用：
演示数据初始化脚本。

关键点：
- 初始化大区、销售员、产品、订单数据。
- 使用相对当前日期的写法，让趋势问答在不同时间运行都能看到结果。

学习建议：
很多“为什么这个问句有结果”都能从这里找到答案。

## 7. `app/logic`：业务与工具层

### `app/logic/agent.py`

作用：
封装真正对外提供能力的 `SalesAgent`。

关键点：
- 提供同步聊天和流式聊天。
- 调用 LangGraph 图来完成一轮对话。
- 处理消息保存、流式输出和图表产物回传。

学习建议：
这是后端里最值得精读的文件之一，因为它连接了“接口层”和“Graph 层”。

### `app/logic/schemas.py`

作用：
定义接口输入输出和业务数据结构。

关键点：
- 聊天请求、登录请求、图表产物、排行结果、趋势结果等都在这里建模。
- 前后端和内部模块之间的数据契约都依赖这些类。

学习建议：
读不懂某个接口该传什么、返回什么时，优先翻这里。

### `app/logic/services.py`

作用：
承接业务规则。

关键点：
- 负责权限过滤。
- 负责缓存。
- 调用 Repository 做销售额、排行、趋势、同比环比等计算。
- `MemoryService` 负责持久化对话记忆。

学习建议：
把它理解成“业务大脑”，它决定同一张表的数据应该怎样被解释。

### `app/logic/tools.py`

作用：
把 Service 能力包装成 Agent 可调用的工具。

关键点：
- 提供确定性的工具函数。
- 负责参数校验、金额格式化、图表 option 生成、异常检测结果格式化。
- 是 Graph 执行节点真正会调用的能力集合。

学习建议：
如果你想知道“用户一句自然语言，最终变成了哪些确定性函数调用”，这里非常重要。

## 8. `app/graph`：Agent 编排层

### `app/graph/__init__.py`

作用：
包初始化文件。

关键点：
- 内容很少，主要是让这个目录成为 Python 包。

学习建议：
知道它的存在即可。

### `app/graph/builder.py`

作用：
构建 LangGraph 工作流。

关键点：
- 把各个节点按顺序连起来。
- 区分同步图和流式图。
- 是“Agent 执行流程图”的真正落地处。

学习建议：
如果你已经大致知道节点干什么，就回来精读这个文件，理解整个链路怎么串。

### `app/graph/context.py`

作用：
定义 Graph 上下文结构。

关键点：
- 当前用户、设置、工具、记忆服务、请求时间等对象都放在这里。

学习建议：
它回答的是“每个节点都能拿到哪些公共信息”。

### `app/graph/helpers.py`

作用：
提供大量 Graph 公共辅助函数。

关键点：
- 构造聊天模型和摘要模型。
- 提供流式 chunk 切分和兜底答案。
- 作为 Graph 里可复用的小型工具箱。

学习建议：
这个文件像“工具箱”，体积大但很实用。精读时建议按功能分段看，不要一口气全啃。

### `app/graph/middleware.py`

作用：
统一收口 Agent 运行期的 trace、状态事件和 token 输出。

关键点：
- 给每次请求挂上 `request_id`。
- 记录 LLM 轮次、工具调用和异常。
- 让节点代码不用重复拼装同样的流式事件。

学习建议：
如果你想知道“流式状态是从哪里发出来的”，这里是关键入口。

### `app/graph/prompts.py`

作用：
集中管理系统提示词。

关键点：
- 包含 ReAct 主提示词和记忆摘要提示词。
- 控制模型该如何理解当前问题、何时调工具、何时声明图表已生成。

学习建议：
如果你想改 Agent 风格或约束输出格式，这里是第一落点。

### `app/graph/router.py`

作用：
定义节点之间的分支路由逻辑。

关键点：
- 判断 Guardrails 之后走哪条路。
- 判断 ReAct 节点之后是继续调工具还是直接结束。

学习建议：
这是“流程分叉条件”的集中位置。

### `app/graph/state.py`

作用：
定义 Graph 运行过程中的状态结构。

关键点：
- 保存问题、计划、工具结果、产物、最终回复等信息。

学习建议：
如果你总是搞不清某个节点输入输出是什么，读这个文件最有效。

## 9. `app/graph/nodes`：Graph 节点实现

### `app/graph/nodes/__init__.py`

作用：
节点目录的包初始化文件。

关键点：
- 主要用于包结构组织。

学习建议：
知道目录里每个文件对应一个节点即可。

### `app/graph/nodes/bootstrap.py`

作用：
初始化或重置图状态。

关键点：
- 在一轮新对话开始时把必要字段补齐。

学习建议：
把它理解成“每轮执行的起跑线”。

### `app/graph/nodes/error_handler.py`

作用：
集中处理 Graph 过程中的错误。

关键点：
- 出错时生成可展示的兜底状态，而不是让整个请求炸掉。

学习建议：
这是提升用户体验很重要但容易被忽略的一层。

### `app/graph/nodes/guardrails.py`

作用：
限制不允许的请求。

关键点：
- 在真正进入 LLM 之前先拦截写操作、预测类请求。
- 被拒绝后会直接结束，不再继续进入 ReAct。

学习建议：
这是“让 Agent 不乱答”的关键措施。

### `app/graph/nodes/memory.py`

作用：
处理会话历史的加载和持久化。

关键点：
- 读取 `sa_chat_memory` 里的整段会话。
- 追问场景会保留最近窗口消息。
- 更早历史会优先压缩成 LLM summary，失败时回退到规则摘要。

学习建议：
如果你关心“多轮对话为什么能记住上下文”，这里要精读。

### `app/graph/nodes/react_agent.py`

作用：
真正调用 LLM 的 ReAct 节点。

关键点：
- 绑定工具 schema 给模型。
- 决定下一步是继续调工具还是产出最终回答。
- 在流式场景下会尽量直接输出模型 token，而不是统一回退到末尾切片。

学习建议：
这是当前 Graph 里最核心、也最值得精读的节点。

### `app/graph/nodes/react_tool_executor.py`

作用：
执行 ReAct 节点请求的工具调用。

关键点：
- 根据工具名找到对应 `StructuredTool`。
- 把工具输出转成 `ToolMessage` 回灌到 ReAct 消息列表。
- 把文本结果合并到 `tool_results`，把图表结果收集到 `artifacts`。

学习建议：
如果你想知道“工具到底是在哪一层真正执行的”，就看这里。

## 11. `frontend/src`：前端源码

### `frontend/src/App.vue`

作用：
Vue 根组件。

关键点：
- 通常只负责承载路由出口和页面级过渡。

学习建议：
业务不多，知道它是壳子就够了。

### `frontend/src/main.js`

作用：
前端真正的 JS 入口。

关键点：
- 创建 Vue 应用。
- 注册 Pinia、Router、Element Plus。

学习建议：
和后端的 `app/main.py` 类似，它负责把整个前端应用装起来。

### `frontend/src/api/index.js`

作用：
前端统一请求封装。

关键点：
- 创建 axios 实例。
- 自动带 token。
- 处理响应和错误。
- 导出 `authApi`、`agentApi`。

学习建议：
想知道前端怎么调用后端接口，就从这里开始。

### `frontend/src/assets/main.css`

作用：
前端全局样式。

关键点：
- 定义整体视觉风格、布局基线、Markdown 展示样式。

学习建议：
如果你改页面样式，这是一个高频入口。

### `frontend/src/components/ChartRenderer.vue`

作用：
图表渲染组件。

关键点：
- 负责接收后端返回的 ECharts option 并绘制图表。
- 处理 resize 和渲染失败场景。

学习建议：
这是“后端图表 JSON -> 前端可视化”的关键桥梁。

### `frontend/src/components/MessageBubble.vue`

作用：
聊天消息气泡组件。

关键点：
- 展示用户消息和 Agent 回复。
- 支持 Markdown。
- 支持图表 artifact 渲染和复制内容。

学习建议：
聊天界面的主要内容展示都在这里。

### `frontend/src/components/MetricsDrawer.vue`

作用：
运行指标抽屉组件。

关键点：
- 从 `/ops/metrics` 拉取运行数据并做展示。

学习建议：
适合学习“后端指标如何被前端消费”。

### `frontend/src/router/index.js`

作用：
前端路由配置。

关键点：
- 定义登录页和聊天页。
- 做基本的登录守卫。

学习建议：
如果你不理解页面切换是怎么发生的，读这里。

### `frontend/src/stores/auth.js`

作用：
登录状态存储。

关键点：
- 保存 token、用户信息、访客模式状态。
- 负责本地持久化。

学习建议：
和后端 `security.py` 配合起来看，理解会更完整。

### `frontend/src/stores/chat.js`

作用：
聊天状态存储。

关键点：
- 管理会话列表、当前会话、消息记录。
- 负责发起同步/流式请求。
- 负责解析 SSE 流事件。

学习建议：
这是前端最值得精读的文件之一，因为它承接了用户输入到界面刷新整个过程。

### `frontend/src/views/ChatView.vue`

作用：
聊天主页面。

关键点：
- 侧边栏、消息区、输入区、快捷问题、指标抽屉都在这里拼装。

学习建议：
如果你想快速把整个前端页面结构看懂，就重点读它。

### `frontend/src/views/LoginView.vue`

作用：
登录页面。

关键点：
- 输入销售员 ID 登录。
- 支持访客模式。

学习建议：
这是前端最简单的页面，适合先热身。

## 13. 脚本目录 `scripts`

### `scripts/perf_probe.py`

作用：
性能探针脚本。

关键点：
- 测同步聊天接口整体耗时。
- 测流式聊天首个 token 的延迟。

学习建议：
你上线前想知道“服务快不快”，用这个脚本。

### `scripts/regression_compare.py`

作用：
Java / Python 结果回归对比脚本。

关键点：
- 对比两边同类工具接口输出是否一致。
- 支持普通文本和图表 JSON 两种比较方式。

学习建议：
这是迁移项目时非常有价值的工具，能帮你理解“如何验证功能没跑偏”。

### `scripts/smoke_test.py`

作用：
烟雾测试脚本。

关键点：
- 快速验证健康检查、同步聊天、流式聊天、图表、异常检测是否能跑通。

学习建议：
每次改完代码先跑它，最省时间。

### `scripts/stability_probe.py`

作用：
稳定性和记忆一致性探针。

关键点：
- 压并发聊天。
- 检查 `sa_chat_memory` 是否真的落库。
- 顺便拉取指标快照。

学习建议：
这个脚本能帮你理解“一个 Agent 项目除了能答题，还要保证状态一致性”。

## 16. 你复现项目时，建议按这个顺序动手

1. 先看 `README.md` 和本文件，知道项目全貌。
2. 检查 `.env` 和 `frontend/.env`，确认数据库、Redis、OpenAI 配置。
3. 跑后端：`run_dev.ps1` 或 `python -m uvicorn app.main:app --reload --port 8088`
4. 跑前端：进入 `frontend` 后执行 `npm install`、`npm run dev`
5. 打开 `/docs` 验证接口。
6. 跑 `scripts/smoke_test.py` 验证主链路。
7. 再开始精读 `app/api/routes.py -> app/logic/agent.py -> app/graph/*`

## 18. 一句话总结每层职责

- `api`：接请求。
- `core`：管配置、日志、鉴权、指标。
- `db`：查数据库。
- `services`：做业务规则。
- `tools`：把业务能力封装成 Agent 工具。
- `graph`：编排 Agent 一轮对话。
- `frontend`：把结果展示给用户。
- `tests/scripts/deploy/docs`：保证项目能测、能跑、能上线、能回滚。
